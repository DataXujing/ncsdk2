    Hardware Network Guidance

    This feature is only available with Intel® Movidius™ Neural Compute SDK 2.x. and neural compute devices based on Intel® Movidius™ Myriad™ X such as the Intel® Movidius™ Neural Compute Stick 2 (Intel® Movidius™ NCS2).

    Overview

    The Neural Compute Engine (NCE) is an on-chip hardware block available in neural compute devices based on Myriad X. It is designed to run deep neural networks in hardware at much higher speeds than possible with previous generations of the Myriad VPU, still with low power and without compromising accuracy. With two NCEs, the Intel® Movidius™ Myriad™ X architecture is capable of 1 TOPS (1 trillion operations per second) of compute performance on deep neural network inferences.

    To take advantage of the hardware acceleration provided by the NCEs, you must compile the neural network to run as a hardware network by setting certain flags for the mvNCCompile tool. See below for compilation instructions.

    Supported Hardware Features:

    Networks utilizing the following supported features can be compiled to run as hardware networks on the NCEs. If your network has other non-hardware features, it can still partially run in hardware on the NCE.

    • Multichannel convolution
      • Matrix-Matrix Multiply/Accumulate
      • Optional non-overlapping Max and Avg pooling
    • Pooling
      • Overlapping Max and Avg pooling
    • Fully connected
      • Vector-Matrix Multiplay/Accumulate
    • Post processing
      • Bias, Scale, Relu-x, PRelu

    Supported Hardware Networks

    Caffe

    • Resnet-18
    • Resnet-50
    • GoogLeNetMx (modified for hardware)
    • GoogLeNet Original
    • VGG-16
    • YoloTiny v1
    • Mobilenet 224
    • SSD Mobilenet
    • Alexnet
    • Squeezenet

    TensorFlow*

    • Inception V1

    Using the Neural Compute API with Hardware Networks

    No application changes are required to use the NCAPI with hardware networks. However, the network graph file must be compiled specifically to run in hardware.

    To compile a hardware network using the mvNCCompile tool, you must set the following flag in the command line:

    • --ma2480

    For most networks you must also set the following flag in the command line (see the release notes):

    • --new-parser

    For some networks, a scaling operation must be applied to each layer for accurate results if you are not using the new parser option (see the release notes). GoogLeNet is a known exception. This can be enabled with the following flag:

    • --accuracy_adjust="<LAYER>:<value>"

    Example usage: --accuracy_adjust="ALL:256"

    • ALL sets value for all layers. If you instead specify the name of a layer it will only apply to that layer. Multiple layers can specified with comma separators.
    • 256 is the value to multiply by (this is an 8-bit shift).
