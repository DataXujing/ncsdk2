    ============================================================

    Movidius Neural Compute SDK Release Notes

    V2.05.00 2018-06-07

    ============================================================

    Important Notes

    Note 1: This version of SDK is compatible with Myriad 2 and Myriad X based devices.

    Note 2: This SDK is pre-release quality. As a result, you may encounter any number of feature limitations or quality issues, including limited documentation or regressions from previous SDKs.

    Note 3: For this release, please use NCS2 MyriadX based device with a USB3 slot or a powered USB2 hub.

    Note 4: For this release, OEM users that build custom firmware need to apply a MDK patch. Please contact Movidius support for guidance.

    SDK Notes:

    New features:

    1. Software or Hardware processing. The Myriad X supports the option of executing networks on shaves, referred to as software processing, or on the CNN engines, referred to as hardware processing. Selection of hardware processing is done during compilation using the –ma2480 flag. Myriad 2 does not support this feature. If this flag is used, hardware compatible layers are run in hardware. (See erratum 2)
    2. In this release the Caffe network parser is being refactored to enable the compiler to do more complex graph optimizations. Some of the networks can only be compiled using the –new-parser flag (see details below). For this release most HW enabled networks will run better using the –new-parser flag. When compiling hardware networks using hand-optimized configurations, users need to associate the appropriate scheduler by using the –scheduler compiler flag.
    3. For hardware capable layers this version of SDK supports automatic compilation. For select networks a hand optimized compilation method is also provided.
    4. TensorFlow 1.7 is now supported.
    5. Using the default parser for some HW networks will have a loss of accuracy due to the CNN engine accumulator. The –adjust_accuracy flag to adjust per-layer accuracy can be used to compensate for accuracy loss (See erratum 16).
    6. Automatic accuracy adjustment has been added when using the –new-parser flag.
    7. The NCSDK API now supports automatic padding and conversion to input tensor format to support CNN engine formats. Apps written for v2.04 or earlier that used HW specific padding should set the HOST_TENSOR_LAYOUT option in the API, or remove the padding/conversion.
    8. Support for fully connected layers that input a 3D tensor.
    9. Experimental Windows support for NCAPI only. See documentation under “Other Installation Options” -> “Windows (NCAPI-Only)” for details.

    Networks:

    1. AlexNet, Inception V1, and SSD-Mobilenet are supported on CNN engines using the –ma2480 and –new-parser flags discussed above.

    Layers:

    API Notes:

    1. API had some refactoring done since last release, please see documentation for the updated API.

    Bug Fixes:

    Errata:

    1. Only Ubuntu 16.04 LTS is supported as a host OS for this release. Raspberry Pi has not been validated.
    2. The SDK includes a GoogLeNet where LRN has been removed and hardware compilation has been set by default, which is found in examples/caffe/GoogLeNetMx. This version has weights that have been retrained on a limited data set. As such, please use the GoogLeNetMx directory only for performance evaluation.
    3. Depth-wise convolution may not be supported if channel multiplier > 1.
    4. If working behind proxy, proper proxy settings must be applied for the installer to succeed.
    5. Non-Overlapping Pooling can run as post operation on HW and as a separate operation in SW. Overlapping pooling is supported as a separate operation on both HW and SW.
    6. Convolution on shaves may fail to find a solution for very large inputs.
    7. Depth convolution is tested for 3x3 kernels.
    8. A TanH layer’s “top” & “bottom” blobs must have different names. This is different from a ReLU layer, whose “top” & “bottom” should be named the same as its previous layer.
    9. Although mvNCCheck shows per-pixel error for some metrics for mobilenet_v1_224 and hardware GoogLeNet, classification results are not impacted.
    10. FC with input NxNxD where N is higher than 1 are not supported natively on CNN Engines. Therefore, they run as a convolution which requires modification of prototxt for this release. Please contact Movidius support for guidance and a helper script.
    11. If the last layer is a reshape it will be optimized out of the network so it will not be reported in the mvNCProfile output.
    12. Simultaneous evaluation of hardware networks is supported in the api which allows an application to double the performance of a particular hardware network. When running hardware processing with two threads the networks must be compiled with 4 shaves or fewer.
    13. A Caffe Scale layer only supports 1 input tensor.
    14. The old Caffe prototxt format is not supported with the new parser. Installation with the nc toolkit will install the correct version. Caffe installations have an upgrade tool “upgrade_net_prototext”. Users who have installed manually may experience issues.
    15. NcProfile will not produce a visual report in this release for generalized parser.
    16. The –accuracy_adjust=VALUES flag should be used if accuracy for HW networks is low when the network is compiled with the default parser. A setting of –accuracy_adjust=”ALL:256” has been found to improve the accuracy for most networks.
    17. Resnet-18, Resnet-50, Inception V1, AlexNet, and SSD-Mobilenet require the –new-parser flag.
    18. For this release if running Alexnet using 2 threads the graph should be compiled with 2 shaves.
    19. Compiling and running a graph with 5 and 15 shaves is not supported.
    20. Tensorflow differs from Caffe in the way padding is done for the tensors. NCE follows the Caffe style and thus would generate different results layer on layer, however this has almost no impact on the full network accuracy and clasification. The impact of this is layer level comparison would fail using mvNCCheck.py.
